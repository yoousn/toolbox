```
                                        ↓

```

## 模型处理方案

**不打包进 exe**,首次使用时程序内下载。

- 下载源: `https://github.com/danielgatis/rembg/releases/download/v0.0.0/u2net.onnx`
- 保存位置: 程序目录下 `.u2net/u2net.onnx`
- 下载器: `backends/model_downloader.py`
  - 带进度条、实时速度、ETA 剩余时间
  - 支持取消、断点保护(.part 临时文件)
  - 可选 sha256 校验
  - 以后加新模型只需在 `MODEL_REGISTRY` 字典加一项

**手动替代方案**: 如果下载太慢,可以从任何渠道下载 `u2net.onnx`,手动放到程序目录的 `.u2net/` 文件夹下,程序会自动检测到。

## 自动更新方案

- 后端: `backends/update_checker.py`
- 启动时后台调用 GitHub API 检查 latest release
- 私人仓库需要 token(环境变量 `TOOLBOX_GH_TOKEN`),Public 仓库不需要
- 版本号只在 `main.py` 的 `__version__` 维护,tag 名与之对应

## GitHub Actions 自动打包(待实施)

在仓库根目录创建 `.github/workflows/build.yml`:

```yaml
name: Build & Release

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: pip install -r requirements.txt pyinstaller

      - name: Build exe
        run: |
          pyinstaller --noconfirm --onedir --windowed ^
            --name "工具箱" ^
            --add-data "qml;qml" ^
            main.py

      - name: Zip output
        run: |
          cd dist
          Compress-Archive -Path "工具箱" -DestinationPath "工具箱-${{ github.ref_name }}.zip"

      - name: Create Release
        uses: softprops/action-gh-release@v2
        with:
          files: dist/工具箱-${{ github.ref_name }}.zip
          generate_release_notes: true
```

注意:不打包 `.u2net/`,模型由用户首次使用时下载。

## 注意事项

- ⚠️ 私人仓库:自动更新检查需要 token,分发给别人用需考虑鉴权问题。如果要分享给同事,建议改 Public。
- ⚠️ 版本号只在 `main.py` 里维护一份,tag 名和它保持一致
- ⚠️ `.u2net/u2net.onnx` 不入 git,已在 `.gitignore` 排除
- ⚠️ 国内访问 GitHub 较慢,模型下载可能需要耐心等待或手动下载放置

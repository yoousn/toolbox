# 工具箱 — 发布与自动更新方案

## 目标

把这个 PySide6 桌面工具箱打包成 exe，发布到 GitHub Releases，实现：
- 任何 Windows 电脑下载 exe 即可运行（无需装 Python）
- 启动时自动检查新版本，有更新弹窗提示一键下载
- 改代码 → push → 自动出新 exe

---

## 整体流程

```
本地改代码 → git push --tags → GitHub Actions 自动打包 exe
                                        ↓
                                发布到 Releases（带版本号）
                                        ↓
用户打开工具箱 → 启动时检查 GitHub latest release → 有新版本弹窗提示 → 下载替换
```

---

## 第一步：项目建仓

在 GitHub 新建一个仓库（比如 `yoousn/my-toolbox`，Private 或 Public 都行），然后把工具箱代码推上去。

```bash
# 本地新建文件夹，把工具箱代码放进去
cd D:\Desktop\my-toolbox
git init
git remote add origin https://github.com/yoousn/my-toolbox.git
git add .
git commit -m "init: 工具箱项目"
git push -u origin main
```

---

## 第二步：添加版本号

在 `main.py` 顶部加一个版本常量：

```python
__version__ = "1.0.0"
```

每次发新版本改这个数字（语义化版本：大改.新功能.修 bug）。

---

## 第三步：GitHub Actions 自动打包

在仓库根目录创建 `.github/workflows/build.yml`：

```yaml
name: Build & Release

on:
  push:
    tags:
      - 'v*'  # 只在打 tag 时触发（如 v1.0.0）

jobs:
  build:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: |
          pip install PySide6 pyinstaller openpyxl onnxruntime Pillow
          # 根据你实际用到的库补充

      - name: Build exe
        run: |
          pyinstaller --noconfirm --onedir --windowed ^
            --name "工具箱" ^
            --add-data "qml;qml" ^
            --add-data ".u2net;.u2net" ^
            main.py

      - name: Zip output
        run: |
          cd dist
          Compress-Archive -Path "工具箱" -DestinationPath "工具箱-${{ github.ref_name }}.zip"

      - name: Create Release
        uses: softprops/action-gh-release@v2
        with:
          files: dist/工具箱-${{ github.ref_name }}.zip
          generate_release_notes: true
```

### 发布新版本的操作

```bash
# 改完代码后
git add .
git commit -m "feat: 新功能描述"
git tag v1.0.1
git push && git push --tags
```

打 tag 后 GitHub Actions 自动跑，几分钟后 Releases 页面就有 exe 的 zip 包了。

## 第四步：程序内自动更新检查

在 `main.py` 启动时加一个版本检查（后台线程，不阻塞启动）：

```python
import urllib.request
import json
import threading

__version__ = "1.0.0"
REPO = "yoousn/my-toolbox"  # 改成你的仓库名


def check_update():
    """后台检查更新，返回 (最新版本, 下载链接) 或 (None, None)"""
    try:
        url = f"https://api.github.com/repos/{REPO}/releases/latest"
        req = urllib.request.Request(url)
        req.add_header("User-Agent", "MyToolbox")
        with urllib.request.urlopen(req, timeout=5) as resp:
            data = json.load(resp)
        latest = data.get("tag_name", "").lstrip("vV")
        if latest and latest != __version__:
            download_url = ""
            for asset in data.get("assets", []):
                if asset["name"].endswith(".zip"):
                    download_url = asset["browser_download_url"]
                    break
            return latest, download_url
    except Exception:
        pass
    return None, None
```

然后在 QML 里弹一个更新提示对话框，点击后用浏览器打开下载链接。

## 注意事项

⚠️ `.u2net/u2net.onnx` 模型文件约 170MB，不要直接 push 到 GitHub（超过 100MB 限制）。两种方案：
  - 用 Git LFS 管理大文件
  - 或者 `.gitignore` 排除它，首次使用时程序自动从某个地址下载

⚠️ PyInstaller 打包时需要把 `qml/` 和 `.u2net/` 作为数据文件包含进去（`--add-data`）

⚠️ 如果仓库是 Private，自动更新检查需要带 token。如果是 Public 就不需要。

⚠️ 版本号只在 `main.py` 里维护一份，tag 名和它保持一致（`v1.0.0` 对应 `__version__ = "1.0.0"`）
